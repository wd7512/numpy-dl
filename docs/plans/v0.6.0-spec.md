### src/numpy_rl/ppo.py (in rl/)
- PPOAgent class:
  - __init__(state_dim, action_dim, hidden_dim=64, lr_actor=3e-4, lr_critic=1e-3, gamma=0.99, lam=0.95, clip_eps=0.2, epochs=10, batch_size=64)
  - actor: Sequential(Dense→ReLU→Dense→Softmax) — policy network
  - critic: Sequential(Dense→ReLU→Dense) — value function
  - select_action(state): sample from actor, get value from critic
  - store_transition(state, action, log_prob, reward, value, done): store trajectory
  - train_step(): compute GAE, PPO clipped update, value loss, entropy bonus
  - train(env, episodes): full training loop
- Creator references:
  - Schulman et al. (2017) "Proximal Policy Optimization Algorithms"
  - Schulman et al. (2015) "Generalized Advantage Estimation"
  - Mnih et al. (2016) "Asynchronous Methods for Deep Reinforcement Learning" (A2C baseline)

  where r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)
  Value loss: L_V = (V_θ(s) - V_target)^2
  Entropy bonus: H = -Σ π(a|s) log π(a|s)

- Shared backbone NOT enforced — separate actor/critic networks (simpler, clearer)
- GAE computed over full trajectory before update
- Multiple PPO epochs per batch of data
